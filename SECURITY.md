# Security Policy

## Supported versions

Security fixes are applied on the `main` branch and released through tagged `slashcash` npm versions.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security reports.

Email security concerns privately to the repository owner via GitHub's [private vulnerability reporting](https://github.com/Ravikumar911/knitly/security/advisories/new) or by opening a draft security advisory if you have maintainer access.

Include:

- A description of the issue and potential impact
- Steps to reproduce
- Affected versions or commit SHAs, if known

We aim to acknowledge reports within a few business days.

## Scope

This project stores personal finance data locally on the user's machine. Reports about misconfigured local credentials, missing `.env` entries, or issues that require physical access to an already-unlocked machine are generally out of scope unless they indicate a remote exploit path.
