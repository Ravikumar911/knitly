# Knitly Personal Spending Analytics Dashboard PRD
**Product Requirements Document for Consumer Spending Insights Platform**

## Overview  
Knitly is a B2C personal finance application that helps individual consumers discover fascinating insights about their spending patterns across different digital platforms and apps. With Swiggy as our flagship merchant integration, we're building a system that reveals hidden patterns, trends, and interesting discoveries about users' spending habits, making personal finance exploration engaging and insightful.

**Problem Statement:** Current dashboard shows generic transaction data without context. Users cannot discover interesting patterns or insights about their spending behavior across different platforms (food delivery, e-commerce, entertainment, etc.). The legacy transaction table v1 is deprecated, and we need to migrate to transactionsV2 with full merchant/app support.

**Value Proposition:** Individual consumers get fascinating insights into their spending patterns that they never knew existed. Users discover their food ordering personalities, spending streaks, seasonal patterns, and unique habits. Each app category reveals detailed behavioral insights that make users say "I didn't know I did that!"

## Core Features  

### 1. App Detection & Classification Engine
- **Automatic App Identification:** Parse transaction data to identify and classify spending apps from transactionsV2.merchantId, merchantCode, and merchantName fields
- **App Categorization:** Classify apps into categories (Food Delivery, E-commerce, Entertainment, Travel, Utilities, etc.) using merchantData JSONB field
- **Spending Intelligence:** Extract app-specific metadata from transaction descriptions and referenceIds

### 2. Dynamic Dashboard Generation System
- **App-Specific Dashboards:** Each app category loads different dashboard layouts and detailed spending insights
- **Swiggy-Focused Dashboard:** Specialized analytics for food delivery spending including order personalities, spending streaks, cuisine adventures, and habit discoveries
- **Detailed Metric Cards:** Rich metric cards that show fascinating spending patterns and behavioral insights
- **Interactive Visualizations:** Engaging charts and graphs that reveal spending stories and patterns

### 3. Detailed Spending Analytics for Food Delivery (Swiggy Focus)
- **Spending Personality Analysis:** 
  - "Weekend Warrior" vs "Weekday Regular" classification
  - "Late Night Snacker" vs "Lunch Champion" profiles
  - "Cuisine Explorer" vs "Comfort Food Lover" categorization
  - "Discount Hunter" vs "Premium Spender" behavior analysis

- **Fascinating Pattern Discovery:**
  - Longest ordering streak (consecutive days/weeks)
  - Most expensive vs cheapest order analysis
  - Peak ordering hour patterns with seasonal changes
  - Weather correlation insights (rainy day orders, sunny day preferences)
  - Payday effect analysis (spending spikes around salary dates)

- **Culinary Journey Insights:**
  - Cuisine diversity score and exploration timeline
  - Restaurant loyalty analysis (favorite spots vs adventure seeking)
  - New restaurant discovery rate and patterns
  - Repeat order percentage and favorite meal analysis
  - Seasonal cuisine preference shifts

- **Time-Based Intelligence:**
  - Detailed day-of-week spending patterns
  - Hour-by-hour ordering heatmaps
  - Monthly spending velocity and trends
  - Year-over-year comparison insights
  - Holiday and special occasion spending spikes

- **Geographic & Delivery Insights:**
  - Home vs office vs other location ordering patterns
  - Delivery radius analysis and comfort zone mapping
  - Neighborhood food preference analysis
  - Travel pattern detection through order locations

### 4. Real-Time Personal Spending Intelligence
- **Live Transaction Processing:** Real-time updates from transactionsV2 with instant insight generation
- **Pattern Recognition:** Automatic detection of new spending patterns and behavioral changes
- **Milestone Celebrations:** Achievement notifications for spending milestones and pattern discoveries
- **Trend Alerts:** Notifications when interesting trends or changes are detected

### 5. Enhanced Transaction Management
- **Legacy Migration:** Complete removal of old transaction table v1 components and migration to transactionsV2
- **Rich Data Enrichment:** Enhance transaction records with behavioral context, timing patterns, and comparative data
- **Smart Pattern Detection:** AI-driven discovery of spending patterns and behavioral insights

## Technical Architecture  

### System Components

**1. Frontend Architecture (apps/main):**
- Remove legacy dashboard components (spending-card-wrapper.tsx, average-spending-card-wrapper.tsx, daily-spending-card-wrapper.tsx, spending-by-day-chart.tsx)
- New insight-focused dashboard framework with engaging analytics
- App-aware routing system (/dashboard/apps/swiggy/insights)
- Rich analytics components with storytelling features

**2. Backend & Database (packages/database):**
- Full migration to transactionsV2 schema with enhanced merchant/app fields
- New insight analytics queries in packages/database/src/queries/insights/
- Pattern recognition engine using merchantData JSONB field
- Real-time analytics computation and caching

**3. API Layer (tRPC):**
- New insight-focused tRPC routers (insights.ts, patterns.ts, analytics.ts)
- Swiggy-specific behavioral analytics endpoints
- Real-time pattern detection and milestone tracking
- Comparative analytics and peer insights APIs

- **The Social Butterfly:** Large group orders, party ordering patterns

- **Value Extremes:** Most expensive vs cheapest order stories

**4. Time Pattern Intelligence:**

- **Seasonal Shifts:** How ordering habits change with seasons
- **Weekly Rhythm:** Detailed day-of-week personality
- **Holiday Behavior:** Special occasion ordering patterns

**5. Geographic & Lifestyle Insights:**

**6. Financial Behavior Discovery:**
- **Spending Velocity:** How spending changes throughout month

**7. Milestone & Achievement System:**

## Risks and Mitigations  

### Technical Challenges
**Risk:** Complex pattern recognition with varied transaction data
**Mitigation:** Build robust pattern matching algorithms with machine learning for behavior analysis

**Risk:** Performance issues with real-time insight computation
**Mitigation:** Implement smart caching, pre-computed insights, and progressive insight loading

### User Experience Risks
**Risk:** Overwhelming users with too much data
**Mitigation:** Progressive disclosure of insights, focus on "wow moments" and digestible discoveries

**Risk:** Privacy concerns with detailed behavioral analysis
**Mitigation:** Clear privacy controls, anonymized comparisons, and transparent data usage
